# Code of Conduct

OSS Maintainer Copilot is committed to a welcoming, respectful, and maintainer-friendly community. We want contributors, users, and reviewers to feel safe asking questions, proposing ideas, and offering feedback.

## Expected Behavior

Participants in this project are expected to:

- be respectful and constructive in discussion
- assume good intent while still giving honest feedback
- focus on the issue, code, or workflow rather than the person
- welcome contributors with different backgrounds and experience levels
- help keep discussions clear, relevant, and actionable

## Unacceptable Behavior

The following behavior is not acceptable in project spaces:

- harassment, discrimination, or hateful conduct
- personal attacks, insults, or intimidation
- sexualized language or unwanted attention
- publishing private information without consent
- trolling, derailing, or intentionally disruptive behavior
- repeated bad-faith arguing after a maintainer decision has been clearly documented

## Scope

This Code of Conduct applies to:

- GitHub issues
- pull requests
- review comments
- repository discussions
- any other project-managed communication channels

## Reporting

If you experience or witness unacceptable behavior, please contact the maintainers privately if possible. If no private path is available, open a minimal issue requesting private follow-up without including sensitive details publicly.

When reporting, include:

- what happened
- where it happened
- who was involved
- any context that would help maintainers investigate

## Maintainer Responsibilities

Maintainers are responsible for clarifying and enforcing this Code of Conduct. They may edit, remove, or reject comments, commits, issues, pull requests, or other contributions that do not align with these standards.

## Enforcement

Project maintainers will review reports fairly and in good faith. Responses may include:

- a private warning
- moderation of conversation
- temporary or permanent restrictions from project spaces

The goal of enforcement is to protect contributors and keep the project healthy, respectful, and sustainable.
